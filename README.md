# spatial2flip

360° / VR 画像をブラウザ上で表示・再生し、ぐるっと一回転させたアニメーション WebP をダウンロードできるシングルページ Web アプリです。

すべての処理はブラウザ内で完結し、画像はサーバーに送信されません。

## 主な機能

- **簡易ビューア**
  - 360° モノラル / VR360 SBS / VR360 OU / VR180 SBS / VR180 OU に対応
  - **Apple 空間写真 (MV-HEIC)** にも対応（iPhone 15 Pro 以降 / Apple Vision Pro で撮影したファイル）
  - WebXR 対応ブラウザ + VR ゴーグル接続時は立体視が可能
  - 非 VR 環境では左目側の画像を表示
  - マウス / タッチドラッグで視点操作
- **動画変換（アニメーション WebP 出力）**
  - 360° 系: カメラを Y 軸周りに一回転させた動画を生成（ステレオ画像は左目視点のみ使用）
  - VR180 / 空間写真: 左右視差を一定間隔で切り替える「パタパタ」アニメーションを生成
  - 回転時間・FPS・解像度・品質を調整可能
- **完全クライアント処理**
  - 画像アップロード・レンダリング・エンコード全て browser 内で実行
  - サーバーへの保存・送信は一切なし

## 技術スタック

| ライブラリ | 用途 |
|---|---|
| [Three.js](https://threejs.org/) r162 | 球面マッピング・WebXR 立体視・オフスクリーン回転レンダリング |
| [ffmpeg.wasm](https://ffmpegwasm.netlify.app/) 0.12.10 | フレーム群を libwebp でアニメーション WebP にエンコード |
| [libheif-js](https://github.com/catdad-experiments/libheif-js) 1.19.8 | Apple 空間写真 (MV-HEIC) のデコード（左右ステレオペア抽出） |
| Python 3 | 開発用静的ファイルサーバー（`serve.py`） |

ビルドステップ不要。素の HTML + ES Modules + CDN で動作します。

## 動作要件

- **ブラウザ**: Chrome / Edge 最新版推奨（WebXR・MediaRecorder・ES Modules をフル活用）
- **Python 3**: ローカル起動用。`file://` では ES Modules と ffmpeg.wasm Worker が動作しないため HTTP サーバー経由が必須
- **ネットワーク**: 初回起動時に CDN から Three.js（約 600KB）と ffmpeg.wasm 一式（約 30MB）をダウンロード。以降はブラウザキャッシュから読み込まれます

## 使い方

### 1. 起動

```bash
python serve.py
# → http://localhost:8000 をブラウザで開く
```

ポートを変更したいときは `python serve.py 8080` のように引数で指定できます。

Python を使わず任意の静的サーバーでも動きます:

```bash
# Node.js の場合
npx serve .

# PHP の場合
php -S localhost:8000
```

### 2. 画像をアップロード

- ドロップゾーンに画像をドラッグ&ドロップ、またはクリックしてファイル選択
- 対応形式: JPG / PNG / WebP / HEIC（Apple 空間写真）
- HEIC ファイルをアップロードすると自動的に「空間写真」フォーマットが選択されます

### 3. 画像形式を選択

| 選択肢 | 想定入力 |
|---|---|
| 360° モノラル | 2:1 のエクイレクタングラー画像（標準的な 360° 写真） |
| VR360 SBS | 左右に 2 枚並んだステレオ 360° 画像 |
| VR360 OU | 上下に 2 枚並んだステレオ 360° 画像 |
| VR180 SBS | 左右に 2 枚並んだステレオ 180° 画像 |
| VR180 OU | 上下に 2 枚並んだステレオ 180° 画像 |
| 空間写真 | iPhone 15 Pro 以降 / Apple Vision Pro が吐く MV-HEIC（通常画角のステレオペア） |

SBS / OU の左右（上下）割り当ては「左 / 上 = 左目」を標準としています。空間写真は MV-HEIC 内の画像 index 0 を左目、1 を右目として扱います。

### 4. プレビュー

ビューアエリアでドラッグ操作。WebXR 対応ブラウザで VR ゴーグルが接続されていれば「ENTER VR」ボタンが有効化され、立体視で鑑賞できます。

### 5. 動画変換

**360° 系（mono360 / sbs360 / ou360）:**

- 回転時間 (1〜30 秒)
- FPS (10〜60)
- 解像度（正方形 or 16:9 プリセット）
- 品質 (WebP 品質 60 / 75 / 90)

→ 「変換開始」でフレームを生成 → アニメーション WebP にエンコード → ダウンロードリンク表示

**VR180 系 / 空間写真（sbs180 / ou180 / spatial）:**

- 切替サイクル数 (1〜20 往復)
- 切替間隔 (0.1〜2 秒)
- 品質

→ 左右画像を交互に切り替えるアニメーション WebP を生成

空間写真は 360° 全天球ではなく通常画角のステレオペアなので、回転動画の生成は行わずパタパタアニメのみ対応しています。

初回の変換では ffmpeg.wasm（約 30MB）の読み込みに時間がかかります。2 回目以降はブラウザキャッシュで高速化されます。

## プロジェクト構成

```
.
├── index.html        # 単一ページ UI
├── css/
│   └── style.css
├── js/
│   ├── app.js        # UI バインディング・状態管理
│   ├── viewer.js     # Three.js ビューア + 360° 回転キャプチャ + 平面ステレオ
│   ├── converter.js  # VR180 / 空間写真のパタパタ生成 + ffmpeg.wasm による WebP エンコード
│   └── heic.js       # libheif-js による MV-HEIC デコード（左右ペア抽出 → SBS 合成）
└── serve.py          # 開発用静的ファイルサーバー
```

## 既知の制約

- ffmpeg.wasm の制約により、出力形式は libwebp によるアニメーション WebP のみ
- 8K 以上の巨大な 360° 画像はブラウザのメモリ制限により扱えないことがあります
- Safari は WebXR 対応が限定的で、立体視機能は Chrome / Edge を推奨
- iOS Safari では一部の MediaRecorder 機能が未対応のため動作しない可能性があります
- **空間写真の制約**:
  - iOS 26 の「空間シーン」（AI 深度による 2D→3D 変換）は非対応。本アプリは本物のステレオペアを持つ **空間写真 (Spatial Photo)** のみ扱います
  - MV-HEIC 内の spatial metadata（baseline / FOV / disparity）は現時点で利用しておらず、FOV は Apple 標準の約 65° をハードコード
  - 360° 回転動画の生成は不可（通常画角の写真のため）
