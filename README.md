# spatial2flip

360° / VR / Apple 空間写真をブラウザ上で表示し、カメラワーク（回転・停止・ズーム・8 の字）を組合せた動画や、左右視差を切り替えるパタパタアニメーションを **アニメーション WebP / MP4 (H.264)** としてダウンロードできるシングルページ Web アプリです。

すべての処理はブラウザ内で完結し、画像はサーバーに送信されません。

## 主な機能

- **簡易ビューア**
  - 360° モノラル / VR360 SBS / VR360 OU / VR180 SBS / VR180 OU に対応
  - **Apple 空間写真 (MV-HEIC)** にも対応（iPhone 15 Pro 以降 / Apple Vision Pro で撮影したファイル）
  - WebXR 対応ブラウザ + VR ゴーグル接続時は立体視が可能
  - 非 VR 環境では左目側の画像を表示
  - マウス / タッチドラッグで視点操作、**マウスホイールで拡大・縮小（FOV ズーム）**
- **カメラワーク詳細設定**
  - 開始位置（視点＋ズーム）を保存し、そこを起点に以下のステップを自由に組合せて動画を生成
    - **回転**: 左右 360° / 180° / 90° / 45°、上下 90° / 45°
    - **ズーム**: ズームイン / ズームアウト（対数空間補間で一定速度のズーム感）
    - **図 8**: 横 8 の字（小 / 大）。開始位置を中心に 1 周して正確に復帰
    - **停止**: 任意秒数その場で静止
  - 各ステップの秒数はインラインで編集可能、合計時間も即時更新
  - ▲▼ ボタン／グリップ `⠿` のドラッグ＆ドロップで並び替え、移動時は行がフラッシュ
  - 「▶ ビューアで再生」で変換前に動きを確認（ビューアが画面外なら自動スクロール）
- **パタパタアニメーション**
  - VR180 / 空間写真で、カメラワーク未設定時は従来どおり左右視差を交互に切り替える動画を生成
  - 切替サイクル数・切替間隔・品質を調整可能
- **出力形式**
  - **WebP**: 無限ループアニメーション（`<img>` 単体で再生、SNS・メッセンジャー互換）
  - **MP4 (H.264)**: yuv420p + faststart で SNS / iOS Safari フレンドリー
  - 変換後に「▶ プレビュー」でインライン再生、ダウンロード可能
- **完全クライアント処理**
  - 画像アップロード・レンダリング・エンコード全て browser 内で実行
  - サーバーへの保存・送信は一切なし

## 技術スタック

| ライブラリ | 用途 |
|---|---|
| [Three.js](https://threejs.org/) r162 | 球面マッピング・WebXR 立体視・オフスクリーンカメラワークキャプチャ |
| [ffmpeg.wasm](https://ffmpegwasm.netlify.app/) 0.12.10 | フレーム群を libwebp / libx264 で WebP / MP4 にエンコード |
| [libheif-js](https://github.com/catdad-experiments/libheif-js) 1.19.8 | Apple 空間写真 (MV-HEIC) のデコード（左右ステレオペア抽出） |
| Python 3 | 開発用静的ファイルサーバー（`serve.py`） |

ビルドステップ不要。素の HTML + ES Modules + CDN で動作します。

## 動作要件

- **ブラウザ**: Chrome / Edge 最新版推奨（WebXR・ES Modules をフル活用）
- **Python 3**: ローカル起動用。`file://` では ES Modules と ffmpeg.wasm Worker が動作しないため HTTP サーバー経由が必須
- **ネットワーク**: 初回起動時に CDN から Three.js（約 600KB）と ffmpeg.wasm 一式（約 30MB）をダウンロード。以降はブラウザキャッシュから読み込まれます

## 使い方

### 1. 起動

```bash
python serve.py
# → http://localhost:8000 をブラウザで開く
```

ポートを変更したいときは `python serve.py 8080` のように引数で指定できます。

Python を使わず任意の静的サーバーでも動きます:

```bash
# Node.js の場合
npx serve .

# PHP の場合
php -S localhost:8000
```

### 2. 画像をアップロード

- ドロップゾーンに画像をドラッグ&ドロップ、またはクリックしてファイル選択
- 対応形式: JPG / PNG / WebP / HEIC（Apple 空間写真）
- HEIC ファイルをアップロードすると自動的に「空間写真」フォーマットが選択されます

### 3. 画像形式を選択

| 選択肢 | 想定入力 |
|---|---|
| 360° モノラル | 2:1 のエクイレクタングラー画像（標準的な 360° 写真） |
| VR360 SBS | 左右に 2 枚並んだステレオ 360° 画像 |
| VR360 OU | 上下に 2 枚並んだステレオ 360° 画像 |
| VR180 SBS | 左右に 2 枚並んだステレオ 180° 画像 |
| VR180 OU | 上下に 2 枚並んだステレオ 180° 画像 |
| 空間写真 | iPhone 15 Pro 以降 / Apple Vision Pro が吐く MV-HEIC（通常画角のステレオペア） |

SBS / OU の左右（上下）割り当ては「左 / 上 = 左目」を標準としています。空間写真は MV-HEIC 内の画像 index 0 を左目、1 を右目として扱います。

### 4. プレビュー

ビューアエリアで:

- **ドラッグ**: 視点回転（360° は全方向、VR180 は前方半球内、空間写真は回転なし）
- **マウスホイール**: 拡大・縮小（FOV 25°〜100° の範囲）
- **ENTER VR**: WebXR 対応ブラウザ + VR ゴーグル接続時に立体視で鑑賞

### 5. 動画変換

**出力形式**: アニメーション WebP（ループ再生）または MP4 (H.264) を選択。

#### 5-A. シンプルモード（ステップ未設定時）

**360° 系（mono360 / sbs360 / ou360）:**

- 回転時間 (1〜30 秒) で Y 軸周り 1 回転
- FPS (10〜60)
- 解像度（正方形 or 16:9 プリセット）
- 品質

**VR180 系 / 空間写真（sbs180 / ou180 / spatial）:**

- 切替サイクル数 (1〜20 往復)
- 切替間隔 (0.1〜2 秒)
- 品質

→ 左右画像を交互に切り替える「パタパタ」アニメーションを生成

#### 5-B. カメラワーク詳細設定モード

360° 系と VR180 で利用可能（空間写真は非対応）。

1. ビューアで**開始位置（視点＋ズーム）**を決める
2. `① 開始位置を設定` ボタンで現在位置を保存（これを押すまで他のステップ操作は無効）
3. 秒数入力の横にあるパレットから好きな動作を追加:
   - 左 360° / 180° / 90° / 45°、右 360° / 180° / 90° / 45°
   - 上 90° / 45°、下 90° / 45°
   - ズームイン / ズームアウト
   - ∞ 小さく / ∞ 大きく（横 8 の字。開始位置を中心に 1 周）
   - 停止
4. 各ステップの秒数は右側の入力欄でいつでも変更可能
5. ▲▼ ボタンまたはグリップ `⠿` のドラッグ＆ドロップで並び替え
6. `▶ ビューアで再生` で変換前にカメラの動きを確認
7. `変換開始` でそのシーケンス通りの動画を生成

VR180 は前方半球しかないので、大きな回転や図 8 大を使うときは**先にズームイン**しておくと半球端で黒背景が見えるのを避けられます。

### 6. プレビューとダウンロード

変換完了後:
- `▶ プレビュー` で WebP は `<img>` に、MP4 は `<video controls autoplay loop>` にインライン表示
- `ダウンロード` リンクから `.webp` / `.mp4` を保存

初回の変換では ffmpeg.wasm（約 30MB）の読み込みに時間がかかります。2 回目以降はブラウザキャッシュで高速化されます。

## プロジェクト構成

```
.
├── index.html        # 単一ページ UI（カメラワーク詳細設定を含む）
├── css/
│   └── style.css
├── js/
│   ├── app.js        # UI バインディング・状態管理（カメラワークのステップ管理、DnD、プレビュー）
│   ├── viewer.js     # Three.js ビューア + カメラワークタイムライン + キャプチャ（compileCameraSteps / evalCameraTimeline）
│   ├── converter.js  # VR180 / 空間写真のパタパタ生成 + ffmpeg.wasm による WebP / MP4 エンコード
│   └── heic.js       # libheif-js による MV-HEIC デコード（左右ペア抽出 → SBS 合成）
└── serve.py          # 開発用静的ファイルサーバー
```

## 既知の制約

- 8K 以上の巨大な 360° 画像はブラウザのメモリ制限により扱えないことがあります
- Safari は WebXR 対応が限定的で、立体視機能は Chrome / Edge を推奨
- MP4 はコンテナレベルでループ情報を持たないため、配布先でのループ再生を期待する場合は WebP を選ぶこと
- **空間写真の制約**:
  - iOS 26 の「空間シーン」（AI 深度による 2D→3D 変換）は非対応。本アプリは本物のステレオペアを持つ **空間写真 (Spatial Photo)** のみ扱います
  - MV-HEIC 内の spatial metadata（baseline / FOV / disparity）は現時点で利用しておらず、FOV は Apple 標準の約 65° をハードコード
  - 平面ステレオなのでカメラワークは利用不可（パタパタのみ）
